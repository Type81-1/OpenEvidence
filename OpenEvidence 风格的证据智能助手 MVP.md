# 暑期实践项目：OpenEvidence 风格的证据智能助手 MVP

## 背景

这次实践课以 OpenEvidence 这类大热的医疗 AI 工具为参考。OpenEvidence 近年快速增长，并进入百亿美元量级估值讨论，说明“专业场景 + 可信证据 + AI 交互”正在成为真实的产品机会。它真正有价值的地方，不是让 AI 直接“给出一个医疗答案”，而是帮助医生或研究者快速找到最相关、最可信、可追溯的文献、临床试验、指南、专利或其它证据。

同时，近期也有研究质疑：专用医疗 AI 工具在一些评测中并不一定优于通用大模型。这说明“做一个 RAG 或医疗助手”本身并不自动代表更专业，关键在于检索质量、引用可靠性、场景设计和效果评估。

本项目希望同学们在 3 天内，用 AI coding 的方式完成一个小型但可演示的 MVP，抓住 OpenEvidence 的核心精髓：不追求做成完整商业产品，而是跑通“问题 → 证据 → 带引用回答 / 对比评估”的关键闭环。

## 项目形式

- **对象**：高年级本科生、低年级研究生，理工科背景为主，有一定编程基础。
- **时间**：共 3 天。前 2 天每个半天为 1.5 小时讲授 + 1.5 小时分组实践；第 3 天全天分组演示和点评。
- **方式**：每组 3～7 人，选择一个赛道，用 Cursor / Codex / Claude Code / Trae / 通义灵码等工具进行 AI 辅助编程。
- **环境**：个人电脑或小服务器 + 在线大模型 API + 公开数据 API。
- **目标**：完成一个可运行原型、若干测试问题、基本评估结果和项目报告。系统不用于真实诊疗，不处理真实患者隐私信息。

## 会涉及什么

同学们会接触到以下内容：

- 医疗/健康场景中的问题建模与产品设计
- PubMed、ClinicalTrials.gov、Europe PMC、公开专利数据库等在线数据资源的 API 使用
- 文献、临床试验、指南、专利、电子书、医学问答样例等资料的采集、清洗和知识库构建
- RAG（检索增强生成）、混合检索、重排、引用溯源
- LLM Wiki 式知识库：把领域资料整理成可读、可维护、可引用的主题知识页
- 幻觉防御，尤其是假引用和检索缺失导致的错误
- Tool / MCP / Skill / 工作流与 Agent 的基本思路
- 通用大模型与专用 RAG 系统的对比评估
- 医疗 AI 的伦理、安全、隐私和责任边界
- AI coding 工程实践、分组协作、演示与报告表达

## 三个可选赛道

### 赛道一：临床证据助手

面向临床医生、医学生或科研人员，聚焦心脑血管病、血脂、高血压、糖尿病等方向。系统需要根据问题检索文献、临床试验、指南和相关证据，并生成带引用的回答。

示例问题：
“高血压患者为什么有时要长期吃药？有哪些指南或研究依据？”
“体检发现血脂偏高，生活方式干预和药物治疗分别有哪些证据？”

### 赛道二：健康营养助手

面向普通消费者，聚焦血脂、三高、饮食干预、营养健康等主题。系统需要把专业文献、公开指南或专业电子书内容转化为可理解、可追溯的健康科普回答。

示例问题：
“地中海饮食对心血管风险有什么证据？”
“限钠饮食对高血压是否真的有帮助？”

### 赛道三：专用 AI vs 通用大模型对比评估

不直接做一个面向用户的助手，而是设计一个小型评测实验，比较“纯通用大模型”和“带领域知识库的 RAG 系统”在同一批医学/健康问题上的表现。

重点问题包括：RAG 是否减少假引用？是否提高完整性？检索质量差时是否反而拖累回答？专用系统到底在哪些场景有优势？

## 前两天辅导内容初步规划

### 第 1 天上午：OpenEvidence 逻辑、场景建模、数据采集与伦理

讲解医疗 AI 的真实使用场景，理解 OpenEvidence 为什么大热，以及“找对文献”为什么比“直接给答案”更稳健；介绍三条赛道、数据来源和项目要求；同时说明医疗 AI 的伦理、安全和隐私边界。

实践任务：确定赛道和分组，设计初版测试问题，使用在线 API 和样例数据启动数据采集。课程可提供一批专业医学问题或问答样例，帮助大家理解真实问法并做系统测试。

### 第 1 天下午：RAG、检索优化与 LLM Wiki 式知识库

讲解 RAG 的基本流程，包括文档切分、向量化、检索、生成和引用展示；介绍混合检索、重排、拒答策略，以及假引用和检索缺失这两类常见幻觉问题。同时介绍 AK 建议的 LLM Wiki 方案：把领域资料整理成主题知识页，作为非 RAG 的知识库路线。

实践任务：完成初版知识库，跑通至少一条带引用回答。

### 第 2 天上午：Tool / MCP / Skill、Agent 与效果评估

介绍如何把 PubMed 检索、引用校验等功能做成工具；说明 Skill、工作流和 Agent 的区别；讲解如何设计测试集、对比纯大模型和 RAG 系统，并评价回答质量。

实践任务：完成至少一项进阶功能或评估实验，跑一批固定测试问题。

### 第 2 天下午：AI coding 工程实践、系统集成与演示准备

讲解如何用 Codex / Claude Code / Cursor / Trae 等工具做小步迭代、调试和协作；指导如何整理界面、演示路径、评估结果和项目报告。

实践任务：完成系统联调，准备第 3 天分组演示。

## 第 3 天产出

每组进行 15～20 分钟展示，内容包括：

- 项目背景和赛道选择
- 系统现场演示或评测结果展示
- 数据来源、方法和核心技术点
- 测试问题与初步评估结果
- 当前局限、伦理边界和后续改进方向
